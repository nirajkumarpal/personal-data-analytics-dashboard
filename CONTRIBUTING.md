# Contributing

Thank you for your interest in contributing to Personal Analytics. This document explains how to contribute, the development workflow, and repo standards.

How to contribute
- Fork the repository and create a feature branch from `main`.
- Open a descriptive PR with clear scope and tests where appropriate.
- Follow the commit message and branch naming guidelines (e.g. `feat/`, `fix/`, `chore/`).

Code style
- Use `black` for formatting and `flake8` for linting. Keep code readable and well-tested.

Testing
- Add unit tests for bug fixes and new features. Aim for clear, deterministic tests.

Review process
- PRs will be reviewed for clarity, tests, and security. Address reviewer comments promptly.
